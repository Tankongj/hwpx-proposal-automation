# Proposal Architect Skill

RFP 분석 결과(Evaluation Matrix)를 바탕으로, 배점에 비례하여 적절한 분량과 목차 구조를 설계합니다.

## 역할 및 책임

"제안서 목차 잡아줘", "구조 기획해줘" 등의 요청 시 다음 작업을 수행합니다.

1. **목차 맵핑**: RFP에서 요구하는 목차(예: Ⅰ. 제안개요, Ⅱ. 제안업체 일반현황 ...)와 평가 항목을 1:1로 매핑합니다.
2. **분량 할당 알고리즘**: 전체 제한 분량(예: 50페이지) 내에서, 배점 비율에 따라 페이지 수를 계산하여 배분합니다. (예: 20점 배점 항목은 전체의 약 20% 분량 차지). 분량이 없는 경우(정성/정량 등) 적절한 상식선에서 배분.
3. **핵심 메시지 도출**: 각 챕터/섹션별로 강조해야 할 핵심 키워드(Win Theme)를 도출합니다.

## Output 구조 예시 (Architecture Design)

```markdown
# 제안서 구조 설계서

## 전체 개요
- 총 예상 페이지: XX p
- 대상 템플릿: `templates/v4_template.hwpx`

## 목차 및 분량 배분

### Ⅰ. 사업수행 부문 (총 X페이지 / 배점 X점)
#### 1. 추진 전략 (X페이지 / 배점 X점)
- **Win Theme**: 디지털 전환 기술과 기존 시스템의 완벽한 융합
- **필수 포함 요소**: 현행 아키텍처 분석, To-Be 이행 방안

### Ⅱ. 일반 부문 ...
```

## 에이전트 지침
- 반드시 `proposal-analyzer`의 결과물인 매트릭스를 참조해야 합니다.
- 배점이 높은 항목은 더 많은 하위 뎁스(Sub-sections)를 가지도록 세밀하게 설계하십시오.
