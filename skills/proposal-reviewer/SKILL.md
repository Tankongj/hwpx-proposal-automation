# Proposal Reviewer Skill

작성된 마크다운 제안서 초안이 HWPX 변환기(`md_to_hwpx.py`)에 들어가기 전 마지막으로 품질과 정합성을 검토하는 과정입니다.

## 역할 및 책임

"작성된 초안 검토해줘", "리뷰 진행해" 등의 요청 시 다음 7-Dimension 검사를 수행합니다.

### 7-Dimension Review Checklist

1. **RFP 정합성**: 평가 매트릭스의 모든 항목이 빠짐없이 목차와 본문에 포함되었는가?
2. **분량 적절성**: 특정 항목이 너무 부실하거나 너무 길지 않은가?
3. **문체 일관성**: 개조식 문체(~임, ~음, ~함)가 아닌 서술형(~입니다, ~할 것이다) 문서가 섞여 있지 않은가?
4. **기호 체계 검증**: `□ → ○ → ― → · → ※` 순서가 역전되거나 이상하게 중첩되지 않았는가?
5. **금지어 검증**: 블라인드 평가일 경우 특정 회사명이나 개인 이름이 포함되지 않았는가?
6. **모호성 제거**: "추진할 예정이다", "노력할 것이다" 등의 표현을 "추진함", "수행함"의 확정적 표현으로 바꿨는가?
7. **논리적 모순**: 앞부분의 예산 단위/일정이 뒷부분과 모순되지 않는가?

## 에이전트 지침
- 오류를 발견하면 사용자에게 "이런 부분이 문제인데 수정할까요?" 라고 묻지 말고, 바로 AI가 해당 마크다운 파일의 내용을 수정(`multi_replace_file_content` 또는 다시 쓰기)해야 합니다.
- 수정한 후 "어떤 항목들을 어떻게 교정했다"고 요약 보고서를 제시하십시오.
