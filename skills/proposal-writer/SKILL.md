# Proposal Writer Skill

`proposal-architect`가 설계한 목차를 바탕으로 뼈대에 살을 붙여 초안(마크다운 형태)을 작성합니다.

## 역할 및 책임

"제안서 내용 작성해줘", "초안 작성 시작" 등의 요청 시 다음 작업을 수행합니다.

1. **콘텐츠 생성**: 레퍼런스(PPT, 이전 제안서 등) 텍스트를 바탕으로 목차 구조에 맞는 개조식 텍스트를 생성합니다.
2. **스타일 규정 준수**:
   - 대주제는 명사형 종결 ("~ 시스템 구축 전략")
   - 본문/하위 항목은 개조식 종결 ("~임", "~음", "~함", "~됨")
   - 모호한 표현(할 수 있다, 예정이다) 지양 및 확정적/단정적 표현 사용
3. **기호 체계 준수**: `□ → ○ → ― → · → ※` 구조로 레벨링하여 텍스트를 생성합니다.

## 작성 규칙 사항 (매우 중요)

`md_to_hwpx.py`는 아래 기호에 반응하여 HWPX 스타일을 맵핑합니다. **반드시 이 포맷을 지켜야 합니다.**

```markdown
# 1장, 2장 등 챕터 (H1)
## 1.1절, 1.2절 등 섹션 (H2)
### 세부 섹션 (H3)

□ 대주제 (가장 큰 카테고리)
  ○ 중주제
    ― 소주제
    ― 소주제
  ○ 중주제
    ※ 참고사항 또는 출처 표기 (우측 정렬용)
```

## 에이전트 지침
- 내용이 너무 짧으면 안 됩니다. 정부 제안서의 특성상 논리적이고 풍부한 서술이 필요합니다.
- 표가 들어가야 할 부분은 마크다운 테이블 문법(`| Title | Title |`)을 활용하여 삽입합니다. (스크립트가 알아서 리스트 형태로 안전하게 폴백(fallback)시킵니다).
